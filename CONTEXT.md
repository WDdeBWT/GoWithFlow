# GoWithFlow 项目上下文

> 本文件记录项目关键决策、当前状态与待办，便于新 AI 快速接手，减少上下文重复。每次完成重要工作后，应更新本文件、提交 git、并部署到远程服务器。

## 项目简介

- **名称**：GoWithFlow
- **目标**：通过自然语言理解用户的“模糊出行意图”，推荐符合组合氛围的周末地点。
- **产品形态**：MVP 先以 Web 应用验证，后续接入微信小程序。

## 关键决策

| 维度 | 决策 |
|---|---|
| 城市 | 仅北京 |
| 交通 | 仅自驾（默认开车），公共交通/骑行写入后续规划 |
| 输入 | 自然语言短句，允许不完整；缺失关键信息时只追问一次 |
| 输出 | 3 个推荐，同时展示列表与地图 |
| 数据 | 本地 JSON 种子 POI 库，MVP 不爬取大众点评/小红书 |
| 反馈 | 底部固定按钮“哪里不对？”，替换全部 3 个推荐 |
| 后端 | Python 3.11 + FastAPI，不使用向量数据库 |
| AI | DeepSeek API（`deepseek-v4-flash`） |
| 天气 | Open-Meteo（无需 API Key） |
| 地图 | 高德 JS API（需 Key，可选；无 Key 降级列表+坐标） |

## 配置

- 后端 `.env`：`DEEPSEEK_API_KEY`、`DEEPSEEK_MODEL=deepseek-v4-flash`、`AMAP_JS_KEY`
- 前端 `.env`：`VITE_AMAP_JS_KEY`（可选）
- 项目提供 `.env.example` 模板；`.env` 已加入 `.gitignore`，不进入版本库。

## 部署

- 远程 Linux 服务器，root 可免密 SSH 登录；需与现有两个 Web 服务隔离。
- 默认端口：后端 `8088`，前端 `8089`。
- 部署前检查端口占用，必要时调整。
- 部署方式优先 Docker Compose；若无 Docker 则使用 systemd。
