# GoWithFlow MVP 产品需求文档

## 1. 项目定位

解决“模糊意图”的周末出行决策：用户用自然语言描述心境，系统推荐符合“组合氛围”的独特地点。MVP 先以 Web 验证，后续接入微信小程序。

## 2. MVP 范围

- **城市**：北京
- **交通**：仅自驾（默认开车），公共交通/骑行写入后续规划
- **输入**：自然语言短句，允许不完整，系统可轻量追问
- **输出**：3 个推荐地点，同时展示列表与地图
- **数据**：本地 JSON 种子 POI 库，MVP 不爬虫

## 3. 核心交互流程

1. **首页**：醒目输入框 + 示例卡片（如“开车去山里，凉快，喝咖啡写代码，顺便爬两步山”）。
2. **轻量追问**：若缺少关键信息，只弹出 1 个追问，避免填表负担。
3. **结果页**：返回 3 个推荐，左侧列表，右侧地图，每个地点附 3 条推荐理由。
4. **反馈替换**：底部固定“哪里不对？”，点击后替换全部 3 个推荐。

## 4. 功能模块

### 4.1 自然语言意图解析器

提取维度：

- **交通半径**：默认开车；将“山里/郊区”映射为约 1–2 小时车程；前端需给出说明并允许用户调整。
- **硬约束**：必须满足的条件，如“有电源插座”“非露天咖啡座”“宠物可进”。
- **软偏好**：环境形容词（“凉快”→温度 < 26℃、“安静”→评论含“人少”）、氛围（松弛/专注）。
- **核心动作**：工作、喝咖啡、徒步、发呆等。

缺失处理：若无法提取硬约束，默认不设限；若缺少关键信息，系统只追问一次。

### 4.2 POI 语义匹配引擎

- **数据源**：本地 JSON 种子库（POI 名称、经纬度、类型、评论摘要）。
- **匹配逻辑**：
  1. 关键词预筛选：根据意图中的动作/环境词缩小候选范围。
  2. DeepSeek API 文本评分：将用户意图与候选 POI 描述送入模型，输出 0–100 的“氛围匹配分”。
  3. 最终排序：`可达性得分 × 氛围匹配度 × 惊喜指数`。
     - **可达性**：Haversine 距离换算预计车程（城市约 30 km/h，高速/郊区约 60 km/h）。
     - **氛围匹配**：DeepSeek 语义评分。
     - **惊喜指数**：POI 描述中低频但正面词的 TF-IDF 得分。
- **去重策略**：若 Top 结果集中在同一商圈，降权并插入周边 5 km 内次级匹配项。

### 4.3 动态环境加权器

- **天气源**：Open-Meteo（无需额外 API Key，提供降水概率、温度、风力）。
- **规则**：
  - 降水概率 > 70%：所有“露天/山野”POI 最终得分 × 0.2，并前置推荐室内替代项。
  - 温度 > 32℃ 且意图含“户外走两步”：优先推荐评论含“溪流/林荫道”的地点。
  - 软偏好与实时天气冲突时，自动降权并附加说明。

### 4.4 结果展示

每个地点附：

- 名称、类型标签、预计车程
- 3 条推荐理由：语义匹配、天气/环境适配、距离成本
- 硬约束标签（如“有电源”“宠物可进”）

地图同步显示 3 个点位，点击标记可定位到对应卡片。

### 4.5 对话修补

- 结果下方固定按钮“**哪里不对？**”。
- 用户点选“太远了/太热了/太吵了/太网红”等选项。
- 系统基于 Top 失败维度，在次级候补池中升维抽取新的 3 个推荐，不重新全量计算。

## 5. 技术方案

### 5.1 后端

- **语言/框架**：Python 3.11 + FastAPI
- **意图解析**：DeepSeek API 返回结构化 JSON
- **POI 排序**：DeepSeek API 对候选 POI 打分
- **天气**：Open-Meteo API
- **数据**：本地 JSON 文件，不引入数据库
- **配置**：`.env` 文件，用户把 Key 粘贴进去即可

### 5.2 前端

- **框架**：React 18 + Vite + Tailwind CSS
- **地图**：高德地图 JS API（需要 `AMAP_JS_KEY`；无 Key 时降级为“列表 + 坐标文本”）
- **定位**：浏览器 Geolocation API，用户拒绝时默认北京天安门坐标

### 5.3 配置示例

`.env.example`：

```ini
DEEPSEEK_API_KEY=your_deepseek_key_here
AMAP_JS_KEY=your_amap_key_here
```

## 6. 部署计划

- **目标**：用户远程 Linux 服务器，需与现有两个 Web 服务隔离。
- **默认端口**：后端 `8088`，前端 `8089`。
- **方式**：优先 Docker Compose；若服务器无 Docker，则提供 systemd 脚本。
- **部署前检查**：确认 8088/8089 是否被占用，必要时调整端口。
- **用户配合**：需临时提供 root 登录方式（SSH 私钥或一次性密码），项目不保存密码。

## 7. 里程碑

1. **Week 1**：北京种子 POI 数据 + DeepSeek 意图解析 + 排序接口
2. **Week 2**：前端页面 + 高德地图 + 天气加权
3. **Week 3**：反馈替换 + 服务器部署 + 调优

## 8. 风险与后续

- DeepSeek API 成本与稳定性
- 高德地图 JS Key 的申请与可用性
- 真实 POI/评论数据后续通过合规 API 接入
